Question 1 — Scalar Strict-Feedback System

Consider the strict-feedback system:

\[ \dot{x}_1 = x_2 + x_1^2, \qquad \dot{x}_2 = u. \]

Step 1 — Virtual Control Design

Define the first error variable:

\[ z_1 = x_1. \]

Choose the first Lyapunov function:

\[ V_1 = \tfrac{1}{2}\,z_1^2. \]

Its time derivative is:

\[ \dot{V}_1 = z_1\,\dot{z}_1 = z_1\bigl(x_2 + x_1^2\bigr). \]

We want \(\dot{V}_1 = -k_1 z_1^2\) when \(x_2\) equals a virtual control \(\alpha_1(x_1)\). Setting

\[ z_1\bigl(\alpha_1 + x_1^2\bigr) = -k_1 z_1^2 \quad\Longrightarrow\quad \alpha_1(x_1) = -k_1 x_1 - x_1^2. \]

Step 2 — Actual Control Design

Define the second error variable:

\[ z_2 = x_2 - \alpha_1(x_1). \]

Note that \(x_2 = z_2 + \alpha_1\), so

\[ \dot{V}_1 = z_1(z_2 + \alpha_1 + x_1^2) = -k_1 z_1^2 + z_1 z_2. \]

Augment the Lyapunov function:

\[ V_2 = V_1 + \tfrac{1}{2}\,z_2^2 = \tfrac{1}{2}z_1^2 + \tfrac{1}{2}z_2^2. \]

Differentiate \(z_2\):

\[ \dot{z}_2 = \dot{x}_2 - \dot{\alpha}_1 = u - \frac{\partial \alpha_1}{\partial x_1}\dot{x}_1 = u - (-k_1 - 2x_1)(x_2 + x_1^2). \]

Compute \(\dot{V}_2\):

\[ \dot{V}_2 = -k_1 z_1^2 + z_1 z_2 + z_2\dot{z}_2 = -k_1 z_1^2 + z_2\bigl[z_1 + u + (k_1 + 2x_1)(x_2 + x_1^2)\bigr]. \]

To obtain \(\dot{V}_2 = -k_1 z_1^2 - k_2 z_2^2\), choose:

\[ u = -z_1 - k_2 z_2 - (k_1 + 2x_1)(x_2 + x_1^2). \]

This yields \(\dot{V}_2 = -k_1 z_1^2 - k_2 z_2^2 \le 0\), guaranteeing global asymptotic stability.

Question 2 — 2-DOF Manipulator Backstepping

Given the manipulator dynamics \(M(q)\ddot{q} + C(q,\dot{q})\dot{q} + G(q) = \tau\), define:

\[ z_1 = q - q_d, \qquad \alpha_1 = \dot{q}_d - K_1 z_1, \qquad z_2 = \dot{q} - \alpha_1. \]

Choose the composite Lyapunov function:

\[ V_2 = \tfrac{1}{2}\,z_1^T z_1 + \tfrac{1}{2}\,z_2^T M(q)\,z_2. \]

Differentiate:

\[ \dot{V}_2 = z_1^T \dot{z}_1 + z_2^T M(q)\dot{z}_2 + \tfrac{1}{2}\,z_2^T \dot{M}(q)\,z_2. \]

Since \(\dot{z}_1 = \dot{q} - \dot{q}_d = z_2 + \alpha_1 - \dot{q}_d = z_2 - K_1 z_1\), we get:

\[ z_1^T \dot{z}_1 = -z_1^T K_1 z_1 + z_1^T z_2. \]

For the second term, note \(\dot{z}_2 = \ddot{q} - \dot{\alpha}_1\), and from the dynamics \(M\ddot{q} = \tau - C\dot{q} - G\), so:

\[ M\dot{z}_2 = \tau - C\dot{q} - G - M\dot{\alpha}_1. \]

Using the skew-symmetry property \(z_2^T(\dot{M} - 2C)z_2 = 0\):

\[ \dot{V}_2 = -z_1^T K_1 z_1 + z_1^T z_2 + z_2^T\bigl[\tau - C\dot{q} - G - M\dot{\alpha}_1 + C z_2\bigr]. \]

Simplifying \(-C\dot{q} + Cz_2 = -C(\dot{q} - z_2) = -C\alpha_1\), so:

\[ \dot{V}_2 = -z_1^T K_1 z_1 + z_2^T\bigl[z_1 + \tau - C\alpha_1 - G - M\dot{\alpha}_1\bigr]. \]

Choose the backstepping torque:

\[ \tau = M\dot{\alpha}_1 + C\alpha_1 + G - z_1 - K_2 z_2 \]

which yields \(\dot{V}_2 = -z_1^T K_1 z_1 - z_2^T K_2 z_2 \le 0\).

Part 1 — Sliding Mode Control Theory

1.1   Motivation and Core Idea

Sliding mode control (SMC) is a variable structure control method. The controller switches between two or more structures depending on which side of a chosen surface the state currently lies. The key benefit is invariance: once the state reaches the sliding surface, the closed-loop dynamics become completely independent of matched uncertainties and disturbances.

Key Insight: SMC decomposes the control problem into two sub-problems: (1) design a surface on which the error dynamics are desirable, and (2) design a control law that drives the state to that surface in finite time.

Figure 1: Phase portrait of sliding mode control showing two phases. Blue trajectories approach the sliding surface (reaching phase), then follow it toward the origin (sliding phase, green). The red line is the sliding surface s = 0.

1.2   Two Phases of SMC

Phase 1: Reaching Phase

The controller drives the system state from any initial condition to the sliding surface \(s = 0\). During this phase the system is sensitive to uncertainties. The reaching phase ends in finite time thanks to the discontinuous switching term.

Phase 2: Sliding Phase

Once \(s = 0\), the state is constrained to the surface. The system dynamics reduce in order by the number of sliding variables. The resulting motion is governed entirely by the surface parameters, not the plant model.

1.3   Sliding Surface Design

Consider a second-order system with tracking error \(e = x - x_d\). Define the sliding surface:

\[ s = \dot{e} + \lambda\, e, \qquad \lambda > 0. \]

When the state is on the surface (\(s = 0\)), the error dynamics become:

\[ \dot{e} = -\lambda\, e \quad\Longrightarrow\quad e(t) = e(0)\,e^{-\lambda t}. \]

The error converges exponentially with time constant \(\tau = 1/\lambda\). For an \(n\)-th order system, we generalise to:

\[ s = \left(\frac{d}{dt} + \lambda\right)^{n-1} e. \]

For example, for a third-order system (\(n=3\)):

\[ s = \ddot{e} + 2\lambda\dot{e} + \lambda^2 e. \]

1.4   Reaching Law

To drive \(s \to 0\) in finite time, we impose a reaching law on \(\dot{s}\). A common choice is the constant + proportional reaching law:

\[ \dot{s} = -\eta\,\operatorname{sign}(s) - k\,s, \qquad \eta > 0,\; k > 0, \]

where

\[ \operatorname{sign}(s) = \begin{cases} +1 & s > 0 \\ 0 & s = 0 \\ -1 & s < 0 \end{cases}. \]

The term \(-\eta\,\operatorname{sign}(s)\) guarantees finite-time reaching, while \(-k\,s\) speeds up convergence when \(|s|\) is large.

Figure 2: Reaching law comparison. (a) Constant rate: s decreases linearly. (b) Exponential + constant: faster initial approach, then finite-time reaching. (c) Power rate: accelerates convergence near the surface for smoother arrival.

1.5   Control Law Decomposition

Consider a general second-order uncertain system:

\[ \ddot{x} = f(x, \dot{x}) + b(x)\,u + d(t), \]

where \(f\) contains known/nominal dynamics, \(b\) is the input gain, and \(d(t)\) is a bounded matched disturbance with \(|d(t)| \le D\).

With \(s = \dot{e} + \lambda e\), differentiate:

\[ \dot{s} = \ddot{e} + \lambda\dot{e} = \ddot{x} - \ddot{x}_d + \lambda\dot{e} = f + b\,u + d - \ddot{x}_d + \lambda\dot{e}. \]

The control law is decomposed as:

\[ u = u_{\text{eq}} + u_{\text{sw}} \]

Equivalent Control \(u_{\text{eq}}\)

Derivation

The equivalent control is obtained by setting \(\dot{s} = 0\) and \(d = 0\) (nominal conditions):

\[ 0 = f + b\,u_{\text{eq}} - \ddot{x}_d + \lambda\dot{e} \] \[ u_{\text{eq}} = \frac{1}{b}\bigl(\ddot{x}_d - \lambda\dot{e} - f\bigr). \]

This term cancels the known dynamics and commands the desired trajectory — it is the smooth, model-based part of the control.

Switching Control \(u_{\text{sw}}\)

Derivation

The switching control handles uncertainties and disturbances:

\[ u_{\text{sw}} = -\frac{1}{b}\bigl(\eta\,\operatorname{sign}(s) + k\,s\bigr), \qquad \eta > D. \]

The gain \(\eta\) must exceed the disturbance bound \(D\) to guarantee reaching.

The complete SMC law is therefore:

\[ u = \frac{1}{b}\bigl(\ddot{x}_d - \lambda\dot{e} - f - \eta\,\operatorname{sign}(s) - k\,s\bigr). \]

Figure 3: SMC block diagram. The controller decomposes into equivalent control u_eq (model-based, maintains system on surface) and switching control u_sw (drives system to surface and rejects disturbances). The sliding variable s is computed from state feedback.

1.6   Existence Condition (Reaching Condition)

For the state to reach and remain on the surface, we need:

\[ s\,\dot{s} < 0 \qquad \forall\; s \neq 0. \]

This is equivalent to requiring that \(s\) decreases in magnitude regardless of its sign. Using a Lyapunov approach, choose \(V = \tfrac{1}{2}s^2\):

\[ \dot{V} = s\,\dot{s} < 0. \]

Lyapunov Stability Proof

Substituting the full control into \(\dot{s}\):

\[ \dot{s} = f + b(u_{\text{eq}} + u_{\text{sw}}) + d - \ddot{x}_d + \lambda\dot{e}. \]

The equivalent control cancels nominal terms:

\[ \dot{s} = d - \eta\,\operatorname{sign}(s) - k\,s. \]

Now compute \(s\,\dot{s}\):

\[ s\,\dot{s} = s\,d - \eta\,|s| - k\,s^2 \le |s|\,D - \eta\,|s| - k\,s^2 = -(\eta - D)|s| - k\,s^2. \]

Since \(\eta > D\) and \(k > 0\), we have \(s\,\dot{s} < 0\) for all \(s \neq 0\). This proves:

  1. The sliding surface is reached in finite time.
  2. Once reached, the state cannot leave the surface.

Finite reaching time: With \(k = 0\) (pure sign), \(\dot{V} = s\dot{s} \le -(\eta - D)|s|\). Since \(V = \frac{1}{2}s^2\), we get \(|s| = \sqrt{2V}\), so \(\dot{V} \le -(\eta - D)\sqrt{2V}\). Solving this differential inequality gives:

\[ t_{\text{reach}} \le \frac{|s(0)|}{\eta - D}. \]

1.7   The Chattering Problem

The discontinuous \(\operatorname{sign}(s)\) function causes the control to switch at infinite frequency near \(s = 0\). In practice, actuator bandwidth is finite, producing chattering — high-frequency oscillations around the sliding surface.

Why chattering is harmful: It excites unmodelled high-frequency plant dynamics, causes mechanical wear, increases energy consumption, and can destabilise the system.

Figure 4: Chattering problem visualization. Left: ideal SMC with infinite switching frequency -- trajectory reaches and stays on the surface. Right: practical SMC with finite actuation bandwidth -- trajectory oscillates around the surface, producing high-frequency control chatter that causes actuator wear.

Solution 1: Boundary Layer (Saturation Function)

Replace \(\operatorname{sign}(s)\) with a continuous approximation inside a boundary layer of width \(\varphi\):

\[ \operatorname{sat}\!\left(\frac{s}{\varphi}\right) = \begin{cases} \displaystyle\frac{s}{\varphi} & |s| \le \varphi \\[6pt] \operatorname{sign}(s) & |s| > \varphi \end{cases} \]

Inside the boundary layer \(|s| \le \varphi\), the control becomes a high-gain linear controller. This eliminates chattering but introduces a steady-state error proportional to \(\varphi\). The trade-off is:

  • Small \(\varphi\): less steady-state error, more chattering.
  • Large \(\varphi\): no chattering, larger steady-state error.

Figure 5: Boundary layer approach for chattering mitigation. Left: phase plane showing the boundary layer band (orange) around the sliding surface (red). Outside the boundary, discontinuous sign(s) is used; inside, smooth sat(s/phi) provides linear control. Right: comparison of sign(s) (discontinuous) vs sat(s/phi) (continuous).

Solution 2: Super-Twisting Algorithm

The super-twisting algorithm achieves finite-time convergence to \(s = 0\) with a continuous control signal (no chattering). It uses an integral of the sign function:

\[ \dot{s} = -k_1\,|s|^{1/2}\,\operatorname{sign}(s) + v, \qquad \dot{v} = -k_2\,\operatorname{sign}(s). \]

Equivalently, the switching control becomes:

\[ u_{\text{sw}} = -\frac{1}{b}\Bigl(k_1\,|s|^{1/2}\,\operatorname{sign}(s) + k_2\!\int_0^t \operatorname{sign}(s)\,d\tau\Bigr). \]

The discontinuity is hidden inside the integrator, so the actual control \(u\) is continuous. Sufficient conditions for convergence are \(k_1 > 0\) and \(k_2 > k_1 D / (k_1 - D)\) (with appropriate scaling).

Figure 6: Super-twisting algorithm comparison. Left: standard SMC produces chattering at the surface with discontinuous control. Right: super-twisting achieves finite-time spiral convergence to the origin with a fully continuous control signal, eliminating chattering completely.

1.8   Multi-Variable Extension

For an \(n\)-DOF system with error \(\mathbf{e} \in \mathbb{R}^n\), define the sliding surface vector:

\[ \mathbf{s} = \dot{\mathbf{e}} + \Lambda\,\mathbf{e}, \qquad \Lambda = \operatorname{diag}(\lambda_1, \dots, \lambda_n),\; \lambda_i > 0. \]

The Lyapunov function becomes:

\[ V = \tfrac{1}{2}\,\mathbf{s}^T \mathbf{s}, \]

and the reaching condition generalises to:

\[ \dot{V} = \mathbf{s}^T \dot{\mathbf{s}} < 0. \]

Part 2 — SMC for Robot Manipulator

2.1   System Model

The \(n\)-DOF rigid-body manipulator dynamics:

\[ M(q)\,\ddot{q} + C(q, \dot{q})\,\dot{q} + G(q) = \tau + d(t), \]

where \(d(t)\) is a bounded disturbance torque with \(\|d(t)\| \le D\). We have nominal estimates \(\hat{M}, \hat{C}, \hat{G}\) and bounded modelling errors:

\[ \Delta M = M - \hat{M}, \quad \Delta C = C - \hat{C}, \quad \Delta G = G - \hat{G}. \]

Figure 7: 2-DOF planar manipulator schematic. Joint angles theta_1, theta_2 are controlled by torques tau_1, tau_2. Link parameters (m_1, l_1, m_2, l_2) are subject to uncertainty (dashed clouds). Gravity g acts downward, and external disturbances d(t) affect the end-effector.

2.2   Sliding Surface

Define the joint tracking error and sliding surface:

\[ e = q - q_d, \qquad s = \dot{e} + \Lambda\,e = \dot{q} - \dot{q}_r, \]

where the reference velocity is defined as:

\[ \dot{q}_r \triangleq \dot{q}_d - \Lambda\,e. \]

This notation simplifies the derivation considerably. Note also:

\[ \ddot{q}_r = \ddot{q}_d - \Lambda\,\dot{e}. \]

2.3   Control Law Derivation

Full Derivation

Step 1: Differentiate \(s\):

\[ \dot{s} = \ddot{q} - \ddot{q}_r. \]

From the dynamics: \(\ddot{q} = M^{-1}(\tau + d - C\dot{q} - G)\), so:

\[ M\dot{s} = \tau + d - C\dot{q} - G - M\ddot{q}_r. \]

Step 2: Rewrite using \(\dot{q} = s + \dot{q}_r\):

\[ M\dot{s} = \tau + d - C(s + \dot{q}_r) - G - M\ddot{q}_r = \tau - M\ddot{q}_r - C\dot{q}_r - G - Cs + d. \]

Step 3: The equivalent control (using nominal estimates, \(d=0\), \(\dot{s}=0\)):

\[ \tau_{\text{eq}} = \hat{M}(q)\,\ddot{q}_r + \hat{C}(q,\dot{q})\,\dot{q}_r + \hat{G}(q). \]

Expanding \(\ddot{q}_r\) and \(\dot{q}_r\):

\[ \tau_{\text{eq}} = \hat{M}(q)\bigl[\ddot{q}_d - \Lambda\dot{e}\bigr] + \hat{C}(q,\dot{q})\bigl[\dot{q}_d - \Lambda e\bigr] + \hat{G}(q). \]

Step 4: The switching control:

\[ \tau_{\text{sw}} = -K\,\operatorname{sign}(s), \]

where \(K = \operatorname{diag}(K_1, \dots, K_n)\) with each \(K_i\) large enough to overcome uncertainty bounds.

Step 5: Complete SMC torque:

\[ \tau = \hat{M}\ddot{q}_r + \hat{C}\dot{q}_r + \hat{G} - K\,\operatorname{sign}(s) \]

Or with the boundary layer:

\[ \tau = \hat{M}\ddot{q}_r + \hat{C}\dot{q}_r + \hat{G} - K\,\operatorname{sat}\!\left(\frac{s}{\varphi}\right) \]

2.4   Lyapunov Stability Proof

Proof of Finite-Time Reaching

Choose \(V = \tfrac{1}{2}\,s^T M(q)\,s\). Note \(V > 0\) since \(M\) is positive definite. Differentiate:

\[ \dot{V} = s^T M\dot{s} + \tfrac{1}{2}\,s^T \dot{M}\,s. \]

Substituting \(M\dot{s} = \tau + d - C\dot{q} - G - M\ddot{q}_r\) and the control law:

\[ M\dot{s} = \hat{M}\ddot{q}_r + \hat{C}\dot{q}_r + \hat{G} - K\operatorname{sign}(s) + d - C\dot{q} - G - M\ddot{q}_r. \]

Grouping terms:

\[ M\dot{s} = -\Delta M\ddot{q}_r - \Delta C\dot{q}_r - \Delta G - Cs + d - K\operatorname{sign}(s), \]

where we used \(C\dot{q} = C(s + \dot{q}_r) = Cs + C\dot{q}_r\). Therefore:

\[ \dot{V} = s^T\bigl[-\Delta M\ddot{q}_r - \Delta C\dot{q}_r - \Delta G + d - K\operatorname{sign}(s)\bigr] + s^T(-Cs) + \tfrac{1}{2}s^T\dot{M}s. \]

By the skew-symmetry property \(s^T(\dot{M} - 2C)s = 0\), the last two terms cancel:

\[ \dot{V} = s^T\bigl[\underbrace{-\Delta M\ddot{q}_r - \Delta C\dot{q}_r - \Delta G + d}_{\text{lumped uncertainty } \delta} - K\operatorname{sign}(s)\bigr]. \]

Let \(|\delta_i| \le \rho_i\) (component-wise bound). Then choosing \(K_i > \rho_i\):

\[ \dot{V} \le -\sum_{i=1}^{n} (K_i - \rho_i)|s_i| < 0, \qquad \forall\; s \neq 0. \]

This guarantees finite-time reaching and invariance of the sliding surface.

Part 3 — SMC for Mobile Robot

3.1   Unicycle Kinematic Model

The unicycle model for a differential-drive mobile robot:

\[ \dot{x} = v\cos\theta, \qquad \dot{y} = v\sin\theta, \qquad \dot{\theta} = \omega, \]

where \(v\) is the linear velocity and \(\omega\) is the angular velocity. The control inputs are \(u = [v,\; \omega]^T\).

Figure 8: Mobile robot SMC tracking. Left: top-down view showing the differential-drive robot tracking a reference trajectory, with tracking errors (e_x, e_y) and polar error rho. Right: sliding manifold in error space showing trajectories converging to the sliding surface then sliding to the origin.

3.2   Position Tracking Formulation

Define the tracking errors in the body frame. Let \((x_d(t), y_d(t), \theta_d(t))\) be the desired trajectory. The position errors in the world frame are:

\[ e_x = x_d - x, \qquad e_y = y_d - y. \]

Transform to the robot body frame:

\[ \begin{bmatrix} e_1 \\ e_2 \\ e_3 \end{bmatrix} = \begin{bmatrix} \cos\theta & \sin\theta & 0 \\ -\sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix} \begin{bmatrix} e_x \\ e_y \\ \theta_d - \theta \end{bmatrix}. \]

3.3   Sliding Surface Design

For position tracking, define two sliding surfaces:

\[ s_1 = \dot{e}_1 + \lambda_1 e_1, \qquad s_2 = \dot{e}_2 + \lambda_2 e_2. \]

An alternative approach for the unicycle (which is non-holonomic and cannot track \(e_2\) independently) is to use a polar coordinate formulation. Define:

\[ \rho = \sqrt{e_x^2 + e_y^2}, \qquad \alpha = \operatorname{atan2}(e_y, e_x) - \theta, \qquad \beta = -\theta - \alpha. \]

Then define sliding surfaces:

\[ s_v = \dot{\rho} + \lambda_v \rho, \qquad s_\omega = \dot{\alpha} + \lambda_\omega \alpha. \]

3.4   SMC Control Law

Using the body-frame error dynamics and the reaching law, the SMC control inputs become:

Derivation

The error dynamics in polar coordinates are:

\[ \dot{\rho} = -v\cos\alpha, \qquad \dot{\alpha} = -\omega + \frac{v\sin\alpha}{\rho}, \qquad \dot{\beta} = -\frac{v\sin\alpha}{\rho}. \]

For \(s_v = \dot{\rho} + \lambda_v \rho\): set \(\dot{s}_v = -\eta_v \operatorname{sign}(s_v)\):

\[ v_{\text{eq}} = \frac{\lambda_v \rho}{\cos\alpha}, \qquad v_{\text{sw}} = \frac{\eta_v \operatorname{sign}(s_v)}{\cos\alpha}. \]

For \(s_\omega = \dot{\alpha} + \lambda_\omega \alpha\): set \(\dot{s}_\omega = -\eta_\omega \operatorname{sign}(s_\omega)\):

\[ \omega = \frac{v\sin\alpha}{\rho} + \lambda_\omega \alpha + \eta_\omega \operatorname{sign}(s_\omega). \]
\[ v = \frac{\lambda_v \rho + \eta_v \operatorname{sat}(s_v / \varphi_v)}{\cos\alpha}, \qquad \omega = \frac{v\sin\alpha}{\rho} + \lambda_\omega \alpha + \eta_\omega \operatorname{sat}(s_\omega / \varphi_\omega). \]
Singularity: When \(\alpha = \pm\pi/2\) (goal directly to the side), \(\cos\alpha = 0\). In practice, clamp \(|\alpha| < \pi/2 - \epsilon\) or switch to a Cartesian controller near the singularity.

Part 4 — SMC for Quadrotor Drone

4.1   Quadrotor Dynamic Model

The 6-DOF quadrotor dynamics separate into translational and rotational subsystems. The position dynamics in the world frame:

\[ m\ddot{x} = (\cos\psi\sin\theta\cos\phi + \sin\psi\sin\phi)\,U_1 + d_x, \] \[ m\ddot{y} = (\sin\psi\sin\theta\cos\phi - \cos\psi\sin\phi)\,U_1 + d_y, \] \[ m\ddot{z} = (\cos\theta\cos\phi)\,U_1 - mg + d_z, \]

where \(U_1\) is the total thrust, \((\phi,\theta,\psi)\) are roll, pitch, yaw, and \(d_x, d_y, d_z\) are disturbance forces. The attitude dynamics:

\[ I_{xx}\ddot{\phi} = (\,I_{yy} - I_{zz})\dot{\theta}\dot{\psi} + l\,U_2 + d_\phi, \] \[ I_{yy}\ddot{\theta} = (I_{zz} - I_{xx})\dot{\phi}\dot{\psi} + l\,U_3 + d_\theta, \] \[ I_{zz}\ddot{\psi} = (I_{xx} - I_{yy})\dot{\phi}\dot{\theta} + c\,U_4 + d_\psi. \]

4.2   Hierarchical SMC Design

The standard approach uses a cascaded (hierarchical) structure:

Outer Loop: Position SMC

Computes desired thrust \(U_1\) and desired roll/pitch angles \((\phi_d, \theta_d)\) from position errors.

Inner Loop: Attitude SMC

Tracks desired roll, pitch, yaw using torque inputs \(U_2, U_3, U_4\).

Figure 9: Quadrotor with hierarchical SMC. Four rotors produce thrust forces F_1-F_4. The outer loop (position SMC) computes total thrust U_1 and desired attitudes. The inner loop (attitude SMC) tracks desired roll, pitch, yaw. Wind disturbances are rejected by the sliding mode controller.

Position Sliding Surfaces

Define position errors \(e_x = x - x_d\), \(e_y = y - y_d\), \(e_z = z - z_d\), and sliding surfaces:

\[ s_x = \dot{e}_x + \lambda_x e_x, \quad s_y = \dot{e}_y + \lambda_y e_y, \quad s_z = \dot{e}_z + \lambda_z e_z. \]

Altitude Control (\(z\)-axis)

Derivation

From \(m\ddot{z} = U_1\cos\theta\cos\phi - mg + d_z\), define \(e_z = z - z_d\):

\[ \dot{s}_z = \ddot{e}_z + \lambda_z \dot{e}_z = \frac{U_1\cos\theta\cos\phi}{m} - g + \frac{d_z}{m} - \ddot{z}_d + \lambda_z\dot{e}_z. \]

Setting \(\dot{s}_z = -\eta_z \operatorname{sign}(s_z) - k_z s_z\) and solving for \(U_1\):

\[ U_1 = \frac{m}{\cos\theta\cos\phi}\bigl[\ddot{z}_d + g - \lambda_z\dot{e}_z - \eta_z\operatorname{sat}(s_z/\varphi_z) - k_z s_z\bigr]. \]

Lateral Control (\(x\)- and \(y\)-axes)

Derivation

From the \(x\)- and \(y\)-dynamics, the SMC computes desired virtual accelerations:

\[ \ddot{x}_{\text{cmd}} = \ddot{x}_d - \lambda_x\dot{e}_x - \eta_x\operatorname{sat}(s_x/\varphi_x) - k_x s_x, \] \[ \ddot{y}_{\text{cmd}} = \ddot{y}_d - \lambda_y\dot{e}_y - \eta_y\operatorname{sat}(s_y/\varphi_y) - k_y s_y. \]

The desired roll and pitch are then extracted:

\[ \theta_d = \arctan\!\left(\frac{\ddot{x}_{\text{cmd}}\cos\psi + \ddot{y}_{\text{cmd}}\sin\psi}{\ddot{z}_{\text{cmd}} + g}\right), \] \[ \phi_d = \arctan\!\left(\frac{(\ddot{x}_{\text{cmd}}\sin\psi - \ddot{y}_{\text{cmd}}\cos\psi)\cos\theta_d}{\ddot{z}_{\text{cmd}} + g}\right). \]

Attitude Sliding Surfaces and Control

Define attitude errors \(e_\phi = \phi - \phi_d\), etc., and sliding surfaces:

\[ s_\phi = \dot{e}_\phi + \lambda_\phi e_\phi, \quad s_\theta = \dot{e}_\theta + \lambda_\theta e_\theta, \quad s_\psi = \dot{e}_\psi + \lambda_\psi e_\psi. \]

The attitude SMC torques:

\[ U_2 = \frac{I_{xx}}{l}\bigl[\ddot{\phi}_d - \lambda_\phi\dot{e}_\phi - \frac{(I_{yy}-I_{zz})}{I_{xx}}\dot{\theta}\dot{\psi} - \eta_\phi\operatorname{sat}(s_\phi/\varphi_\phi)\bigr], \] \[ U_3 = \frac{I_{yy}}{l}\bigl[\ddot{\theta}_d - \lambda_\theta\dot{e}_\theta - \frac{(I_{zz}-I_{xx})}{I_{yy}}\dot{\phi}\dot{\psi} - \eta_\theta\operatorname{sat}(s_\theta/\varphi_\theta)\bigr], \] \[ U_4 = \frac{I_{zz}}{c}\bigl[\ddot{\psi}_d - \lambda_\psi\dot{e}_\psi - \frac{(I_{xx}-I_{yy})}{I_{zz}}\dot{\phi}\dot{\theta} - \eta_\psi\operatorname{sat}(s_\psi/\varphi_\psi)\bigr]. \]

Lab Activities

Activity 1 — Chattering Observation (10 min)

Objective: Observe and quantify the chattering phenomenon, then mitigate it using the boundary layer.

  1. Run the manipulator SMC simulation with use_sat = false (pure sign(s)). Plot the control torque \(\tau\). Observe the high-frequency switching.
  2. Switch to use_sat = true and test three boundary layer widths: \(\varphi = 0.01,\; 0.1,\; 1.0\).
  3. For each value of \(\varphi\), record: (a) peak torque oscillation amplitude, (b) steady-state tracking error \(\|e\|_\infty\).

Questions to Answer

  • How does boundary layer width \(\varphi\) affect chattering amplitude?
  • How does \(\varphi\) affect steady-state tracking accuracy?
  • Is there an optimal \(\varphi\)? What criteria would you use to select it?

Activity 2 — Disturbance Rejection Comparison (15 min)

Objective: Compare the robustness of SMC against other control methods under large disturbances.

  1. Add a disturbance torque \(d(t) = 10\sin(5t)\) to the manipulator dynamics.
  2. Run four controllers:
    • SMC with \(\operatorname{sign}(s)\) and \(K = 15\)
    • SMC with \(\operatorname{sat}(s/\varphi)\), \(\varphi = 0.05\), \(K = 15\)
    • Robust control (from Week 4)
    • Computed torque (from Week 1)
  3. Compare tracking errors and control effort. Plot all on the same axes.

Questions to Answer

  • Which controller maintains the smallest tracking error under the disturbance?
  • What happens to the computed torque controller — why does it degrade?
  • How does the switching gain \(K\) relate to the disturbance bound \(D = 10\)?

Activity 3 — Super-Twisting Algorithm (5 min)

Objective: Implement the super-twisting algorithm and compare with standard SMC.

  1. Replace the switching term with the super-twisting law. The integral state \(v\) requires an extra state variable per joint.
  2. Compare the control torque smoothness and tracking error against sign-based and sat-based SMC.
  3. Verify that chattering is eliminated while finite-time convergence is preserved.

Key Takeaway: Sliding mode control provides the strongest robustness guarantee among all six methods for matched disturbances — once on the sliding surface, the system is completely invariant to matched uncertainties. The price is chattering, which can be mitigated using boundary layers or the super-twisting algorithm at the cost of slight performance degradation.
